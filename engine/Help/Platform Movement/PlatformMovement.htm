Platform Movement object

This document refers to an extension made for Multimedia Fusion, by Olle Fredriksson (Fodo), in 2003.

 

What does the object do?

The Platform Movement object was made to replace the simple, in built platform movement in Multimedia Fusion. Now, afterwards, I see it more like a base for a custom platform engine. The cool thing is that it's totally customisable - if you want! You can also use in built premade functions for everything, or get down on a low level manipulating velocities and variables.

 

Usage

The first thing you will see after placing the object in your frame is the dialog, with lots of values to tinker with. I suggest you don't bother too much about it from the beginning, I think it's simpler to just code in the needed events first, then tinker with all values and the presets to get your platform movement the way You want it.

So, for now, just leave it and click Ok.

One big difference between this object and the in built movements is that for this object, you need to use another object as the detector object - it will not work good with a character that animates and varies in size. So, create a detector object, for an example an object with a filled box, about the size of your visible animation object. If you want, make your detector invisible at the start of the frame (either by events or in the object properties).

If you haven't already, create some obstacle backdrops, and possibly some actives with a qualifier to work as jump through platform.

The first thing you need to do in the events editor is to tell the Platform Movement object what object it should operate and move around. Do this with the Set object action. Select your detector. If you have an animation object, always set it to position (0,0) from the detector object.

• Start of Frame
: : Set object to

 

The extension SDK is quite limited when it comes to detecting collisions from an extension, it's not possible at all, unless you use the workaround I've used in this object. The way this works is a bit more complex, and it requires some work from the user - but it gives you all of the work back in flexibility.

When the extension needs to test for an obstacle collision, it triggers it's own condition Test for obstacle overlap, and it's in this event, you are supposed to test for an obstacle overlap and tell the object that the object is overlapping an obstacle.

So, what you do, is that you create an event with the Test for obstacle overlap condition, from the Platform Movement object, then, under that line, test if the detector is overlapping a backdrop. If this event is true, use the action Object does overlap with obstacle so that the extension knows that a collision occured. If you have more than one type of obstacles, maybe an active object, or a group of active objects that function as obstacles, you have to make a new Test for obstacle overlap function for each of these types, and return that the object does overlap.

If you want the object to stop when it leaves the play area, make a new Test for obstacle overlap event and test if the detector is not in the play area.

      Events
 
• : Test for obstacle overlap
+ is overlapping a backdrop
: : Selected object overlaps an obstacle
 
• : Test for obstacle overlap
+ is in the play area
: : Selected object overlaps an obstacle

The next thing you want to do is adding left/right key movement. So make two events - repeat while pressing right, and left, and use the actions in the extension - User is holding right input key, and same with left.

      Events
 
• Repeat while Moved right
: : User is holding right input key
 
• Repeat while Moved left
: : User is holding left input key

 

There! You now have a working platform movement! You probably want jumping, too.

To add jumping, you use the same procedure, but with the Jump action, and you should also add the condition Object is standing on ground from the extension.

If you want higher jumps when the user is holding the jump key, make a new event, Repeat while pressing jump key, and from the extension Object is jumping, and then use the extension's in built User is holding jump in the air function.

      Events
 
• Pressed fire 1
+ : Object is standing on ground
: : Jump
 
• Repeat while Pressed fire 1
+ : Object is jumping
: : User is holding jump in the air

Variables

All variables, except max step up and downhill slope correction are measured with 1/100th of a pixel accuracy. That means that speed 200 is 2 pixels per frame.

Variables can be used for your own custom movement types and additions to your movement. If you want to add ladders, moving platforms and such, you must use the values.

All variables can be changed in the event editor, but the ones categorized as Setup box can also be changed in the setup box. You can apply presets of the setup box variables from the setup box, those are loaded from an ini file named PlatformMovPresets.ini, in the extension directory. If you open that ini file in notepad you will see instructions on how to add your own presets for use in the object.

 

Setup box

  • Max X velocity - the X velocity will automaticly be set to this value if it's higher than it.
  • Max Y velocity - same as above but on the vertical axis.
  • X acceleration - how much to accelerate when moving left and right.
  • X deceleration - how much to decelerate each frame when not moving left or right.
  • Gravity - how much to add to the Y velocity when the selected object is not standing on the ground.
  • Jump strength - what the extension sets the Y velocity to when using the Jump action.
  • Jump hold strength - how much to take from the Y velocity when holding the jump key in the air.
  • Max step up - how many pixels the object can walk up (steps) without stopping.
  • Downhill slope correction - how many pixels down the extension will look for obstacles to "stick" to when close to the ground.

Event editor only

  • X velocity - horizontal speed.
  • Y velocity - vertical speed.
  • Additional X velocity - horizontal velocity that isn't affected by any other variables, use this for for an example moving platforms and magnet effects.
  • Additional Y velocity - same as above but on the vertical axis.


Additional notes

Moving platforms can be tricky to get working fully. There are many ways to achieve this, but the way I do it, is that the moving platform has a custom movement, preferably using the same 1/100th of a pixel accuracy as the extension for it's speed. When you've got your platform, make a Test for jump through overlap event, testing the platform you just made, then, use the Selected object overlaps a jump trough platform, but also, set the additional X velocity of the extension to the speed of the platform. You also have to disable the movement, set additional X velocity to 0, do this in a negated Object is standing on ground event, from the extension. Also do it in the Test for obstacle overlap event, for security. Example:

      Events
 
• : Test for jump through platform overlap
+ is overlapping
: : Selected object overlaps a jump through platform
: : Set additional X velocity to 200
 
• : Object is standing on ground
: : Set additional X velocity to 0
 
• : Test for obstacle overlap
+ is overlapping a backdrop
: : Selected object overlaps an obstacle
: : Set additional X velocity to 0

 

Ladders can be done in many different ways, too. The simplest way is probably to test if the selected object overlaps with the ladder object, and the user is pressing the up key, then set additional Y velocity to -100 or another negative value, also set the maximum Y velocity to 0. Set additional Y velocity to 0 when the user is not pressing the up key, and set the maximum Y velocity to it's default value again. Also do the same when pressing down on a ladder.

If you need to stop the movement completely, Pause movement does this, the selected object will be stopped until you use the Unpause movement action.

Springs are very easy to add. If the selected object overlaps with your spring object, set the Y velocity to a negative value.

 

Animations is something I wanted to have automated... But I found no good way to do this, so you will have to code in the animations with events. But don't see this as a bad thing, it actually gives you more freedom. Remember that you can compare all variables from the extension using Compare two general values under the Special object in Multimedia Fusion.

Here is a small example on animation... also don't forget setting the direction of the animation object to right and left when the user presses right and left. And most important, remember to always place the animation object relative to the detector object.

      Events
 
• : Object is standing on ground
+ : Object is moving
: : Change animation sequence to Stopped
 
• : Object is jumping
: : Change animation sequence to Jumping
 
• : Object is falling
: : Change animation sequence to Falling
 
• : Object is standing on ground
+ Abs( GetXVelocity( " " ) ) > 70
: : Change animation sequence to Walking

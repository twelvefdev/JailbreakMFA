Conditions
Is word in Ignore list?
Condition is true when the given parameter is in the internal list of words to ignore. Words can be ignored using the "Ignore Word" action.
Is word Correct?
Condition is true when the given parameter is in the wordlist or in the ignore list. Note that hyphenation and contraction settings will be considered.
Is loading in background?
Condition is true when a load is being preformed in the background. A background load is triggered by the "Load Wordlist in Background" action.
On Loading
Condition is triggered periodically whilst a background load is happening. This condition is helpful for uploading a "Loading" progress bar (etc)
On Correction Suggestion
Condition is triggered once for each spelling correction suggestion to the word given in the "List Suggestions" action.

Use the "Get Suggestions" action to determine what the correction actually is.
Is Parse in Progress
This condition is true when a parse is in progress. See the general documentation for more information about what this means.
On Incorrect Word
This condition is triggered when an incorrect word is found in an Autoparse.

Note that when this condition occurs the parse is automatically stopped. The "Resume Parse" action should be used to continue.
On Parse Done
This condition is triggered when a parse has completed. This is when the whole text has been checked for spelling mistakes. If the "Stop Parse" action was used, this does not get triggered.
On Parse Aborted
This condition is triggered when a parse has aborted using the "Stop Parse" action.
Actions
List Suggestions
Lists spelling suggestions for the given word. For each correction found, "On Correction Suggestion" will be triggered.

Note that:
  • Only words in the word list are considered. Words in the ignore list are not.
  • Contracted words and hyphenated words will not be listed among the suggestions.
  • The words will be returned will respecting the ordering and suggestion settings.
Load Wordlist (Pauses MMF)
Loads a word list. Using this action will cause your program to freeze until it has completed. This makes it quicker and suitable for small dictionaries, however for loading large wordlists the "Load Wordlist in Background" action should be used. This function will fail if ran whilst a dictionary is being loaded in the background.

Words in the word list can be separated by any non-alpha character.
Load Wordlist in background
Loads a wordlist in the background. Only one wordlist may be loaded in the background at one time. If this action is called whilst another dictionary is loading in the background then it will fail.

After calling this action "On Loading" will be periodically triggered. Additionally, "Get Loading Percentage" allows the percentage to be retrieved at any time.

Words in the word list can be separated by any non-alpha character.
Save Word List (Using null separators)
Saves the currently loaded wordlist as a "Null Separated Word List". This is where each word in the list is separated by the "Null Character", which is a hidden character. For large wordlists, the size of this format will be significantly lower than the "Using New Line Separators" variation of this action.
Save Word List (Using new line separators)
Saves the currently loaded wordlist, separating each word with a new line. (Note that these are Windows style new lines)
Add Word
Adds a word into the wordlist. Note that wordlists (including any wordlists loaded using the "Load" functions) are global and shared between every Spellcheck object, and global between frames.
Remove Word
Removes a word from the wordlist. Note that wordlists (including any wordlists loaded using the "Load" functions) are global and shared between every Spellcheck object, and global between frames. If the word is not currently in the wordlist, this action has no effect.
Ignore Word
Ignores a word, such that "Is Word correct?" does not detect it as an incorrectly spelt word. This can be used to implement "Ignore All" buttons in spellcheckers.
Unignore Word
Unignores a word, removing it from the Ignore List.
Set Suggestion Settings
This action modifies how the "List Suggestions" action works.

The first parameter sets the minimum relevance that the word must be the queried word to be suggested. This is a percentage of how similar the two words are.

The second parameter decided the maximum number of words that can be returned by "List Suggestions". Setting this to -1 will allow unlimited.
Set Ordering Settings
This action modifies how the "List Suggestions" action works. Specifically, it modifies the order in which the words are suggested.

It is beyond the scope of this document to explain what exactly that parameters do, however by playing around with the values (perhaps with the help of the example), you can see which values you prefer.
Set Contraction and Hyphen mode
This action modifies which words "Is Word Correct?" accepts as correct by modifying how hyphenated words and contraction are treated.

See the tutorial style introduction to this object for more information on what the different types mean.
Add Base
This adds a contraction base to the Contraction Base List, for when the Contraction mode is "Selected Bases".

See the tutorial style introduction to this object for more information on what this means.
Remove Base
This removes a contraction base from the Contraction Base List, for when the Contraction mode is "Selected Bases".

See the tutorial style introduction to this object for more information on what this means.
Add Contraction
This adds a contraction ending to the list of contraction endings, for when the Contraction mode is either "Selected Bases" or "Any bases".

Contraction must be either in the form X'X (e.g. n't), 'X (e.g. 's) or 'XX (e.g. 've).

See the tutorial style introduction to this object for more information on what this means.
Remove Contraction
This removes a contraction ending to the list of contraction endings, for when the Contraction mode is either "Selected Bases" or "Any bases".

See the tutorial style introduction to this object for more information on what this means.
Start
Starts an Autoparse. The parameter is the text to parse. If an Autoparse is already happening when this action is executed, nothing will happen.

See the tutorial style introduction to this object for more information on the Autoparse features.
Resume Parse
Resumes the Autoparse after it has been paused. The Autoparse is automatically paused after the "On Incorrect Word" event is called.

See the tutorial style introduction to this object for more information on the Autoparse features.
Stop Parse
Stops the Autoparse. "On Parse Aborted" will be triggered.

See the tutorial style introduction to this object for more information on the Autoparse features.
Change Last Incorrect Word
When "On Incorrect Word" is triggered, you may replace the last incorrect word in the text with a new word by using the "Change Last Incorrect Word" action. To get the correct string after the parse is done, use the "Get Corrected String" expression.

See the tutorial style introduction to this object for more information on the Autoparse features.
Add Ignore Key
Adds a key to the list of keys that cause the parser to get into "Ignore mode". To remove this key, you must use "Reset all parse settings".

See the tutorial style introduction to this object for more information on the Autoparse features.
Add Spellcheck Key
Adds a key to the list of keys that cause the parser to get into "Spellcheck mode". To remove this key, you must use "Reset all parse settings".

See the tutorial style introduction to this object for more information on the Autoparse features.
Add Escape Char
Adds a character to the list of characters that cause the parser to get into "Escape mode". To remove this character, you must use "Reset all parse settings".

See the tutorial style introduction to this object for more information on the Autoparse features.
Add Quote Char
Adds a character to the list of characters that cause the parser to get into "Quote mode". To remove this character, you must use "Reset all parse settings".

See the tutorial style introduction to this object for more information on the Autoparse features.
Set Neutral and General Settings
Sets various Autoparse advanced features.

See the tutorial style introduction to this object for more information on the Autoparse features.
Reset all parse settings
Resets all parse settings (those options settable in this menu)

See the tutorial style introduction to this object for more information on the Autoparse features.
Expressions
Get Contraction Mode
Gets the current contraction mode.

0 = None
1 = Selected Bases
2 = Any Base
Get Contraction Mode
Gets the current contraction mode.

0 = None
1 = Selected Bases
2 = Any Base
Get Maxs Results
Gets the maximum number of results that "List Suggestions" might return.
Get Min Relevance of Results
Gets the minimum relevance necessary for "List Suggestions" to return a word.
Find how similar two strings are
Given two strings, find how similar they are to each other. This is effected by the "Set Ordering Settings" action (and dialog box options).

This is a percentage, where higher is nearer (i.e. 100% = Exactly the same)
Get Suggestion
Returns the last suggested word, after a "On Correction Suggestion" event.
Get Loading Percentage
Returns a percentage value indicating how near to competition a background load is.
Get Incorrect Word
Gets the incorrect word which caused the "On Incorrect Word" event.
Get Incorrect Words Start Index
Returns the index in the string that the incorrect word which caused the "On Incorrect Word" event is.
Get Incorrect Words Length
Returns the length of the incorrect word which caused the "On Incorrect Word" event.
Get Corrected String
Returns the string passed to the "Start Autoparse" action. However, any incorrect words which were replaced with the "Change Last Incorrect Word" action are accounted for. That is, this returns the corrected string.

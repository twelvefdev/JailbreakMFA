The Common Dialog object gives you control of the save and load common dialog that is used by most Windows programs. The Common Dialog gives you control over several properties in how the dialog behaves, and allows you to open a dialog window using an action or expression. You can also set the initial directory, initial filename, use custom filters, and more.

The ability to use custom filters is probably the biggest advantage of using the Common Dialog object. A filter is a filename expression that allows you to limit the files that show up in a user's open or save dialog window where the files and folders are listed. For example, the filter "*.txt" would show only files with the file extension txt, usually called Text files. Another popular filter might be "*.exe", which would show all Executable files. You can add these filters to your dialog box by calling Add Filter before opening the dialog window. For every filter you add a new selection will show up in the Filters listbox in your open or save dialog window. If you want more then one filename extension for a single filter (single entry in the listbox) then you can use the pipe character "|" to separate filter names and filter extensions. For example, you could call the action:

Add Filter "Extension files" for "*.cox|*.gox"

And you will get a single filter that filters for both the cox and gox filename extensions.

Features
  • Flexible adding of custom filters to open and save dialog boxes

  • Can limit selection of files to a single intial directory

  • Can limit users to existing folders and/or files

  • Enable or disable create new file and/or overwrite existing file prompts

  • You can be notified whether the Open/Save or Cancel button was selected

  • You can open the open/save dialog box from an expression and have the filename selected returned in-line without using any actions

Action
Add Filter

Syntax: Add Filter szFilterName for szFilterExt
Comments: Adds the specified filter to the internal filter list, which are used to filter which file types to display in the open and save dialog boxes. You can select between the various filters.
Example: Add Filter "Text files" for "*.txt"
Arguments:
Argument Type Description
szFilterName string A name describing an extension filter. For example, a good name for the extension *.exe would be Executable Files.
szFilterExt string A filter extension, with wildcards. A filter extension of *.dll would look for all files with the three character extension dll.

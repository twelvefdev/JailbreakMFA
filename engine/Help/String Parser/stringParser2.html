The String Parser 2 object gives the developer access to powerful parsing and tokenizing functions to tear apart or put together strings and substrings of information. The parsing and substring functions allow you to separate important pieces of information, and to search for known or even unknown, using the wildcard support, substrings. The wide assortment of tokenizing functions can make dealing with complicated sequences of information a snap.

A list is a string of tokens separated by delimiters. A token is simply a substring that is separated from other tokens/substrings. The delimiter(s) are what do the separating. String Parser 2 supports any number of delimiters, all of which can be of varying length, and can support wildcard matching in wildcard search mode.

Here is an example where I may want to count the number of words in a sentence...

List example, we'll take the following sentences:

Sean was here. Hey guys! See how we can spell D-E-L-I-M-E-T-E-R-S... wait--did you hear that?

So what I'm going to do is say that my delimiters are as follows:
" "
"."
","
"!"
"?"
"--"

That's a total of five delimiters I have to set for these particular sentences. If you were to do this for a whole document be sure to include other punctuation such as ";", ":", etc.

Now if I ask for ListGetAt$("String Parser", 3") I'll get "here" Notice how you get the 3rd word? Cool huh? How about if we get a little trickier and enable wildcards, then we can ask for ListFind$("String Parser", "?-E*", 1) and we'll get 11, because the 11th token, "D-E-L-I-M-E-T-E-R-S" is the first (1) token to match this wildcard expression. And here is the complete list of tokens (words!) you have with those delimiters listed above:
1) "Sean"
2) "was"
3) "here"
4) "Hey"
5) "guys"
6) "See"
7) "how"
8) "we"
9) "can"
10) "spell"
11) "D-E-L-I-M-E-T-E-R-S"
12) "wait"
13) "did"
14) "you"
15) "hear"
16) "that"

If you do a ListCount("String Parser") you'll get 16 ;)

Features
  • Just about all of the features of the old String Parser + lots more

  • No limit to the amount of text that can be stored or manipulated.

  • Loading and saving to file (text, CSV, INI, MMF Array, Dynamic Array)

  • Case sensitive and case insensitive string comparison modes

  • Easy enable/disable switch of wildcard or literal search modes, wildcard supports:
    • ? will match a single character
    • * will match zero or more characters
  • No limit to the number of delimiters that can be used for list tokenizing

  • No limit to the length of a delimiter that can be used for list tokenizing

  • New URLEncode function and Is URL Safe condition to format arguments for a URL (web page)

  • Fifteen List tokenizing functions! Including: ListFind, ListContains, ListSortAsc, etc.

  • Wildcard support in list delimiters for all list tokenizing functions

  • Export and import of Comma-Delimited text files (CSV) which can be imported and exported by Excel, StarCalc, ODBC, etc.

  • Export and import of MMF 1.5 Array files, MMF Pre-1.5 array files can be imported

  • Export and import of Dynamic Array 2 files

  • New feature: Import and export to INI file format

  • Get the MD5 signature of the source string to use as verification or as a checksum

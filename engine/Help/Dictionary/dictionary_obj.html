Dictionary Object

The Dictionary Object is a specialized object that allows you to do very fast lookups to see if a word exists in a very large collection. For example, if you have 100,000 words stored in a dictionary, you wouldn't want to use a normal array to have to search through an average of 150,000 words just to find one. Even doing a logarithmic search would be time consuming, so the Dictionary object uses a large hash table.

Using the object is fairly simple. You can add words individually using Add word and remove them using Remove word, or you can load them from multiple sources (MMF Array, CSV, INI, DA2, etc.). Once you have a word list you can use the condition Word exists to test to see if the word exists or not. If your hash table is large enough you should get an immediate response, without having to wait for the object to parse through large amounts of data. You will get your answer quickly and effeciently.

The most notable applicable use would be a spell-checker. You can download a CSV with a list of common English words here:
English.zip (0.5MB): http://www.3ee.com/docs/examples/english.zip

Features
  • Store an unlimited number of words

  • Import and export to MMF Array format

  • Import and export to CSV (comma-delimited text) file format

  • Import and export to INI file format

  • Import and export to Dynamic Array 2 format

  • Re-sizable hash table to tweak performance (speed vs. memory)

  • Case sensitive and case insensitive string comparison modes

  • Easy enable/disable switch of wildcard or literal search modes, wildcard supports:
    • ? will match a single character
    • * will match zero or more characters

Action
Set table size

Syntax: Set table size to nTableSize
Comments: Sets the dictionary hash table size. Because of how the dictionary stores elements, you will generally get better performance by having a large table size (although there isn't a lot of reason to increase it beyond approximately 220000). A smaller table size will use less memory, but may affect performance. A table size of 1 will use the minimum amount of memory but will also result in the poorest performance, especially for the Word exists condition. If you are using case insensitive mode you will probably want to limit to this no more then 110000 to avoid wasting memory.
Example: Set table size to 50000
Arguments:
Argument Type Description
nTableSize integer The size of the table.

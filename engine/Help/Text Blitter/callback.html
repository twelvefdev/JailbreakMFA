Callback

Introduction

Callbacks are a widely used method within professional programming. Callbacks allow you to take control and alter how certain events are handled. Callbacks can range from controlling huge to very snall amounts of data and are applyable to most situations.

The use of callbacks within the Text Blitter extension are primarily to control how the extension draws itself within your game or application.

The process which controls the callbacks are very similar to those used in the 'Fast-Loop' functions within Multimedia Fusion. The difference here is that, instead of handling loop iterations, you handling a specific task.

[ NOTE: Do NOT use the standard actions when handling a callback condition, you will change the internal data that is currently being used to draw the text and it is mostly likely you will crash the application or corrupt the data. (This is runtime only, we dont want you making applications that crash now do we :P)]

[ NOTE: Callback will NOT work when any animtaions are set. The system is designed to be a replacment for the animation system and cannot work together with it. ]

Along with the callbacks are specific actions and expressions that give you the power to change properties before they are used to draw the text to the screen.

No longer a Text Blitter...

For the clever ones among you, you will notice that having total control over how the text is drawn to the screen gives you the power to turn the extension into what could be called an Image Blitter. You now have the control to choose where on the screen you wish to position the character and where on the character map you wish to grab the image, you can control how wide and how high the image will be, and how much transparency will be used.

Info

Below is a quick reference to what each of the callback conditions do and how you can take control using them.

  • On Begin: is called after the initialisation of the drawing has completed, just before the extension prepares to draw the first line. This event is used best for reseting outside variables that monitor the text in the extension and how everything is being drawn. There are currently no specific actions or expressions for this event as there is nothing you can change.

  • On Begin Line: is called after the line has been set up and is about to draw the first character. This event is very generic and with the help of a couple of actions and expressions you can create special effects by moving the lines around.

  • On Character: is called just before the character is being drawn, from this event you have total control over what is displayed, from the destination positions, source positions to the transparency of the image. This event in the most powerfull.

  • On End Line:, is called once the line has finished being drawn, just before the extension is about to move onto the next line. This event has no actions or expressions but gives you a pointer to when the line is finished for use with other things within your application

  • On End: is called when the object has finished drawing its text and about to exit from the drawing routine, this allows to you know that drawing has completed and run any specific tasks.

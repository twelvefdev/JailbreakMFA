Set Transparency

The transparency of the object alters how Multimedia Fusion draws the object in the frame. When transparency is turned on, the transparent colours selected on the image show as transparent and can be seen through. However, when transparency is turned off, the transparent areas appear as black on the screen.

Set Text Functions

It is sometimes hard to include new lines and tab characters in your text especially when it is stored in a compact format. Text functions allow the text to contain special characters which when drawn are converted to the correct counterparts.

Below, the table outlines what special characters represent which symbols

\n - A new line
\t - A tab character

Word Wrapping

This excellent new feature standard to the extension is known as Word Wrapping and takes the strain away from making sure long pieces of text appear on the screen correctly. As standard, when Word Wrapping is enabled any words that may be cut-off by the edge of the extension's boundary will be wrapped to the next line

The word wrapping engine also compensates for the spaces that should appear the beginning of the new line. This however can cause problems if you want your line to start with a space, the simple work around is to use another space character or turn word wrapping off

Multiline

Sometimes you may wish to prevent the object from moving onto multiple lines, you can do it from these set of actions. When multiline is enabled the extension freely creates multiple lines for both the correct characters and when word wrapping (word wrapping does not work when multiline is disabled)

Load Image

Loads a new image from the hard drive. The action also alters the image width and image height values as it is loaded

1: Filename of the new image

Saves whatever image is loaded into the extension into a file.

Set Image Width

This action does not alter the actual image in any way. It alters how the engine blits the characters from the image to the screen by setting a horizontal boundary. When the engine reaches this theoretical width, it wraps around to the next line to try and get the desired character form there

1: New Width value

Set Image Height

Similar to the action above but has very little impact on the drawing of the text, in most cases the engine will ignore this option.

1: New Height value

Set Image X Offset

When the engine searches the image for the desired character, this value moves the result in the specified amount horizontally (in pixels). So if the character is positioned at 0 (zero), and the argument is 20, the result character will be positioned at 20 on the image.

1: New X location of the offset

Set Image Y Offset

Similar to the action above, except that it works on the y axis. See above for how it works

1: New Y location of the offset

Set Character Width

The Width of the character alters both how the text is drawn to the screen and how the engine searched the image for the desired character. Obviously if the character width is greater, it will take up more space on the screen. However, as the character size increases, the engine uses more space on the image and wraps more characters around to the next line. Please check the section on how the extension works for more information

1: New Width of character

Set Character Height

Similar to the action above, except that it works with the height of the character. Please check the section on how the extension works for more information

1: New Width of character

Set Character Offset

The Character Offset is a value that alters the result character printed to the screen. If for example, the letter 'A' is contained within the extension's text buffer and the offset value is set to the value 2, (if all settings are correct), then the character that will be printed will be 'C'.

1: New Offset

Set Character Map

The character map is simply a string specifying the order of the characters in the image. This action allows you to change this string at runtime. Please check out the section of how the extension works for more information

1: A String containing the new character map

Set Horizonal Spacing

The horizontal spacing is the number of pixels between each character printed to the screen.

1: New Horizontal Space

Set Vertical Spacing

The vertical spacing is the number of pixels between each character printed to the screen.

1: New Vertical Space

Set Horizontal Alignment

The horizontal alignment changes how the text is drawn to the screen. The value specified for this option must be either 0, 1 or 2. 0 specifies that the text should be aligned to the left, 1 to the centre and 2 to the right. Any other value will act the same as using 0

1: Option

Set Vertical Alignment

The vertical alignment changes how the text is drawn to the screen. The value specified for this option must be either 0, 1 or 2. 0 specifies that the text should be aligned to the top, 1 to the centre and 2 to the bottom. Any other value will act the same as using 0

1: Option

The Scrolling options allow you to move the text around inside the object without affecting how it is laid out. Due to the way the engine works any text that goes out of the visible area will not be 'drawn' or spent time thinking about, it is simply skipped and scrolling will not slow down the extension is any way (its as fast as it being static)

Change Alterable Text

Changes the text within the extension. Using Multimedia Fusion 1.2 and below you will be limited to 1024 characters internally, however, if you are using MMF 1.5 or above you can increase the buffer size of the extension (it's in the setup dialog box) and use whatever size you want (WARNING: The larger the buffer, the more memory will be used, the number of characters is the same as the number of bytes occupied. So 1024 characters is basically 1k of memory)

1: String of text

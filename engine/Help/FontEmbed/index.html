Font Embed Object

Documentation

This extension object is super-straightforward, so if you have any problems not covered in this documentation, then you should consider yourself unworthy of human contact and go live as a hermit. Go on; go!

Otherwise, read below.

Overview

The Font Embed Object registers non-local fonts to your application using the Windows API function AddFontResourceExA, found in GDI32.dll.

This enables you to bundle a font with your application and use it without installing the font on the target user's computer. The font will remain private to your application only, so programs like MS Word will be unable to access your font, even if your application is running at the time.

Conditions

There are no conditions. So there. That was simple wasn't it?

Actions

Action Description
Embed Font Registers the font. Write the full path to the font on the hard disk when prompted. You can register multiple fonts at the same time by separating them with a single | pipe character. If you have embedded a font file into your project as a Binary, you'll need to extract it and use the BinFileTempName$() function to locate it.
Remove Font This removes the font. This must contain the same path that you used in the Embed action.

Expressions

Expression Description
Number of fonts added
E.g.
FontsAdded( str ObjectName ) 		Returns the number of fonts registered by the last call Embed action. If 0, then the embed failed.

Supported Font Types

The Windows API function used by this extension supports the following file types:

.fon, .fnt, .ttf, .ttc, .fot, .otf, .mmm, .pfb, .pfm

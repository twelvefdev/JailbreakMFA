The Dynamic Array is a full-featured dynamic array that supports realtime multiple dimensioning, lightweight database-esq features, storing values, strings, and files, and more.

Although an array can always be viewed as a 1-dimensional structure, DA2 allows you to create a multi-dimensional view of the array. This means you can access DA2 as if it were a 2D, 3D, 4D, up to 10D array. It provides simple conversion expressions to convert a 1D coordinate like X=23 to a 3D coordinate like "4,5,3" (which is in the form "X,Y,Z").

In a multi-dimensional array you will set constraints, called dimension boundaries for every dimension below the dimension of the array. For example, if you have a 3D array you must set the dimension boundaries for dimension 1 (X) and dimension 2 (Y). This means that X will have a range, Y will have a range, and Z will be able to grow or shrink dynamically as you add and remove elements to the array.

Simple database-like features exist for 2D Arrays, which are referred to as Recordsets. Dimension 1 is considered to be the columns (or fields) and dimension 2 then consists of the rows. Since dimension 1 is fixed (has a dimension boundary of some set size that you must specify, for example 10 elements), you will have a fixed number of columns, but a variable number of rows.

Basic Features
  • Just about all of the features of the old Global Array + lots more

  • Store mixed strings, values, and binary files as elements in the array

  • Import and export to MMF Array format

  • Import and export to CSV (comma-delimited text) file format

  • Import and export to INI file format

  • Define the array to be 1-dimensional up to 10-dimensional

  • You can resize dimension boundaries (max X, Y, etc.) dynamically during runtime

  • Associative keys in the form of value and string keys (mixed OK!)

  • Quick sorts for ascending and descending order using strings or values

  • List functions of elements, keys, or paired

  • Array aggregate functions (max, min, sum, avg)

  • Simple array searches based on matching substrings

  • Expressions for converting between n-Dimensional indices and the equivalent 1-dimensional index

  • Case sensitive and case insensitive string comparison modes

  • Easy enable/disable switch of wildcard or literal search modes, wildcard supports:
    • ? will match a single character
    • * will match zero or more characters


These advanced features will apply if you have defined a 2D array, referred to as a Recordset.

Advanced Features (Database)
  • Mixing of Inclusive (OR) and Exclusive (AND) SELECT modes supported for filtering

  • SELECT WHERE clause (LIKE, =, <, <=, >, >=, <>, BETWEEN) filtering

  • Aggregate querying (MIN, MAX, SUM, AVG)

  • Ability to associate labels (fieldnames) to columns (fields)

  • Rows can be represented by lists

  • Simple row sorting by column

Get capacity

Syntax: capacity(szObject)
Comments: Returns the current capacity of the array. This is the number of elements allocated, even though many of the elements may not be in use. Since the array is dynamic, it will allocate more memory as it needs it, and it attempts to grow and shrink the capacity in a fairly intelligent manner to minimize the number of memory re-allocations.
Example: capacity("Dynamic Array")
Arguments:
Argument Type Description
szObject string Name of object.

Get dimension boundary

Syntax: boundary(szObject, nDim)
Comments: Returns the boundary of the specified dimension. You should have set boundaries for every dimension less then the dimension of the array. So for a 3D array you should set dimension boundaries for dimension 1 and 2 (X and Y constraints, where Z is free to grow or shrink).

Argument Type Description
szObject string Name of object.
nDim integer The number of dimensions. A grid or matrix is a 2-dimensional structure, whereas the x,y,z coordinate system is a 3-dimensional coordinate system.

Syntax: Resize dimension nDim boundary to nBoundary
Comments: Converts the existing dimension boundary to a new dimension boundary. Resize extension boundary may move data, insert empty columns, and/or delete existing data in order to preserve the location of existing elements. If you simply want to change the dimension boundary without having any data moved then you should use Set dimension boundary instead. You should set up a boundary for every dimension that is less then the dimension of your array. So if you have a 4-dimensional array you should have boundaries for dimension 1, 2, and 3.
Example: Resize dimension 1 boundary to 25
Arguments:
Argument Type Description
nDim integer The number of dimensions. A grid or matrix is a 2-dimensional structure, whereas the x,y,z coordinate system is a 3-dimensional coordinate system.
nBoundary integer The maximum index number for a particular dimension. For example if you want 10 columns then you might set boundary 1 to 10 in a 2D array. You could then have as many rows as you needed, each with 10 columns.

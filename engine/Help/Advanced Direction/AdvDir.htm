Advanced Direction Object Documentation

The Advanced Direction Object is an extension written for Multimedia Fusion 1.5 by Olle Fredriksson (also known as Fodo) in 2003.

 

Function List

Conditions

Compare Distance between two Points - If the distance between the two points is lower or equal to the third parameter input by the user, it returns true. You can negate this condition (right click the condition and select Negate) to "reverse it's logics". That means if the distance is greater than the third parameter, it returns true, when negated.

Compare Direction between two Points - If the direction between the two points is equal to the direction input by the user, it returns true. The third (last) parameter also lets you give the extension offset values. If the direction between the two points is closer than that to the direction input by the user, then it returns true. You can negate this condition (right click the condition and select Negate) to "reverse it's logics".

Actions

Set Number of Directions - This sets the number of directions for the extension to use with all functions. It is also possible to set this value in the setup dialog box of the object. You can use whatever number you want. If you're using Active Objects, it might be good to use 32 directions, or with Active Picture, it might be good with 360. Please note that almost all functions will be affected by a direction change.

Get Object Distances and Fixed Values - This function first lets you pick an object, and then a position. The extension will loop through all objects that meet the current condition's demands, and get their Fixed Values and their distances to the point. Then you must use the expressions Get Fixed and Get Distance to get objects. Scroll down a bit to get to those functions.

Expressions

Get Number of Directions - This gets the current number of directions that the extension uses. Use Set Number of Directions or the dialog box to change this value.

(Stored Objects)

Get Fixed - Gets the Fixed Value of an object gotten from the Get Object Distances and Fixed Values action. The parameter is a value which tells the extension to get a certain object. Using 0 gets the closest object to the point (also from the Get Object... function), 1 the second closest object, 2 the third object, etc.

Get Distance - Gets the distance of an object from Get Object... to the point, also from Get Object... just like the Get Fixed function, this gives you the closest object if you use 0, and second closest if you use 1, etc.

 

(Distance and Direction between two Points)

(Float Values (decimals))

Direction between two Points - This expression is affected by the current number of directions. The first parameter is the first x coordinate, the second is the first y coordinate, third the second x, and fourth the second y ( x1, y1, x2, y2 ). This expression returns decimal numbers. Not for use with other functions in the extension because most of them don't support decimals.

Distance between two Points - The first parameter is the first x coordinate, the second is the first y coordinate, third the second x, and fourth the second y ( x1, y1, x2, y2 ). This expression returns decimal numbers. Not for use with other functions in the extension because most of them don't support decimals.

(Long Values (no decimals))

Direction between two Points - This expression is affected by the current number of directions. The first parameter is the first x coordinate, the second is the first y coordinate, third the second x, and fourth the second y ( x1, y1, x2, y2 ). This expression doesn't give decimal numbers. For use with other functions in the extension.

Distance between two Points - The first parameter is the first x coordinate, the second is the first y coordinate, third the second x, and fourth the second y ( x1, y1, x2, y2 ). This expression doesn't give decimal numbers. For use with other functions in the extension.

 

(Difference between two Directions)

Absolute - Gets the difference between two directions (affected by the number of directions). This expression will give positive numbers only.

Not Absolute - Gets the difference between two directions (affected by the number of directions). This expression will sometimes return negative numbers.

 

(Get Movement)

Get X Movement from a Speed and a Direction - Affected by the number of directions. Useful for making movements with irregular number of directions. The first parameter is the direction, and the second is the speed. It gives you the value of how much something would move on the x axis with the given direction and speed. This function will return decimal values.

Get Y Movement from a Speed and a Direction - Same as Get X Movement... but on the y axis.

 

Rotate - Gives you a direction calculated from the three parameters. The first parameter is the current direction. The second parameter is the direction to smootly rotate to. The third parameter is the rotating speed. The function will return the current direction after it has been rotated towards the direction to rotate to with the rotating speed.

Convert Direction to a different Base - Converts a direction to a direction with a different base. First parameter is the direction (using the current number of directions), and the second parameter is the number of directions you want the direction to have. For an example, this is useful if you're using 360 directions for the movement of an Active Object. Then you can convert the movement directions to the base-32 directions of the Active Object!

 

 

Usage

The object provides a set of functions that are very useful when creating custom movements, and doing other movement related things.

If you try to get an object's fixed value or distance with a value that is out of the object range (a negative value, or a value that is greater than the number of objects stored in the extension) the extension will give you the farthest away object from the point.

Imagine you're making a Real Time Strategy type game. What will you use the extension for? First of all you want smooth, non looping movement and rotating of your Active Object units. So you use a 360 direction movement, using the Get Movement functions, and use Convert to a different Base to set the direction of them. When you've got the movement down, you probably want to detect enemies, to see if they are in range of your units. Then loop through the units (or unit turrets if you're using them), and on loop, you get the enemy unit objects with Get Object... and use the position of the friendly unit. Now you can get the first object's (0) distance to see if it's in range - if it is in range, you want your unit shooting at it...!
